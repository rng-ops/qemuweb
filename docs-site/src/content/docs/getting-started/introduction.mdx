### The Three-Layer Architecture

1. **Main Thread (UI Layer)**
   - React application with terminal UI (xterm.js)
   - `QemuClient` API for VM control
   - AI agent integration (LangGraph)

2. **Web Worker (Execution Layer)**
   - Runs QEMU WebAssembly in an isolated thread
   - Handles VM lifecycle and I/O
   - Communicates via `postMessage` protocol

3. **Storage Layer (Persistence)**
   - Block devices backed by IndexedDB
   - Copy-on-Write overlays for efficient snapshots
   - Content-addressed storage (Atlas Store)

## Key Features

### üöÄ Zero Installation
No need to install Docker, VirtualBox, or any other software. If your browser supports WebAssembly and SharedArrayBuffer, you can run VMs.

### üíæ Persistent Storage
VMs maintain their state across browser sessions using IndexedDB. Changes are stored efficiently using Copy-on-Write, so you only store what's changed.

### üîí Sandboxed Security
Each VM runs in a completely isolated Web Worker. There's no access to your host filesystem, network, or other browser tabs.

### ü§ñ AI Integration
Built-in support for AI agents via LangGraph and Model Context Protocol (MCP) servers. Let AI control your virtual infrastructure!

### üåê Virtual Networking
Configure complex network topologies with multiple VMs, routers, and firewall rules‚Äîall running in your browser.

## Use Cases

- **Education**: Learn Linux, networking, or system administration without affecting your real system

- **AI Training**: Create controlled environments for training AI agents

## Browser Requirements

<Aside type="caution">
  QemuWeb requires specific browser features that need proper server configuration.
</Aside>

| Requirement | Details |
|-------------|---------|
| WebAssembly | Core technology for running QEMU |
| SharedArrayBuffer | Required for multi-threaded WASM |
| Cross-Origin Isolation | Headers: `COOP` and `COEP` |
| IndexedDB | For persistent storage |

See [Browser Requirements](/qemuweb/getting-started/requirements/) for detailed compatibility information.

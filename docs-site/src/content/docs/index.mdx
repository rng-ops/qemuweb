## Why QemuWeb?

<CardGrid stagger>
  <Card title="Zero Installation" icon="rocket">
    Run complete Linux VMs directly in your browser. No Docker, no VirtualBox, no admin rights needed. Works on any modern browser.
  </Card>
  <Card title="WebAssembly Powered" icon="laptop">
    QEMU compiled to WebAssembly for near-native performance. Leverages SharedArrayBuffer and multi-threading for optimal speed.
  </Card>
  <Card title="Persistent Storage" icon="document">
    Copy-on-Write block devices with IndexedDB backing. Your VM state survives browser refreshes and sessions.
  </Card>
  <Card title="AI-Driven Infrastructure" icon="star">
    Built-in LangGraph agent with MCP servers for AI-controlled virtual environments and autonomous operations.
  </Card>
</CardGrid>

## Quick Example

```typescript
import { QemuClient } from '@qemuweb/runtime';
import { profiles } from '@qemuweb/vm-config';

// Create a VM client
const client = new QemuClient({
  workerUrl: '/worker.js',
  wasmUrl: '/qemu-system-x86_64.wasm',
});

// Start an Alpine Linux VM
await client.start(profiles.alpineLinux);

// Send commands to the VM
client.sendSerial('echo "Hello from the browser!"\n');

// Handle VM output
client.on('serial', (data) => {
  console.log('VM output:', data);
});
```
